    **2. `fakturowniaApiRequestAllItems`**
    ```typescript
    export async function fakturowniaApiRequestAllItems(
      this: IExecuteFunctions,
      method: IHttpRequestMethods,
      endpoint: string,
      body: IDataObject = {},
      qs: IDataObject = {},
    ): Promise<IDataObject[]>
    ```
    Implementation:
    - NOTE: This function reads `returnAll` and `limit` at index 0. This is the standard n8n pagination convention -- the pagination helper is called once per execute() invocation, not per input item. The caller in execute() loops items and calls this once for list operations. This is intentional and matches the official n8n pattern (e.g., Wise.node.ts).
    - `const returnAll = this.getNodeParameter('returnAll', 0) as boolean`
    - `const limit = returnAll ? 0 : (this.getNodeParameter('limit', 0) as number)`
    - `const perPage = 100` (max allowed by Fakturownia)
    - Loop: `let page = 1; const allItems: IDataObject[] = []`
    - Each iteration: call `fakturowniaApiRequest.call(this, method, endpoint, body, { ...qs, page, per_page: perPage })`
    - Push response items into allItems
    - If `!returnAll && allItems.length >= limit`: return `allItems.slice(0, limit)`
    - If `response.length < perPage`: break (last page detected)
    - Increment page
    - Return allItems

    Import types from 'n8n-workflow': `IDataObject`, `IExecuteFunctions`, `IHttpRequestMethods`, `IHttpRequestOptions`, `NodeApiError`, `JsonObject`.
