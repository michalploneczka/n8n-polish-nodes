```typescript
import { NodeConnectionTypes, type INodeType, type INodeTypeDescription } from 'n8n-workflow';

export class Nbp implements INodeType {
    description: INodeTypeDescription = {
        displayName: 'NBP Exchange Rates',
        name: 'nbp',
        icon: 'file:../../icons/nbp.svg',
        group: ['transform'],

        defaults: { name: 'NBP' },
        usableAsTool: true,
        inputs: [NodeConnectionTypes.Main],
        outputs: [NodeConnectionTypes.Main],
        // NO credentials - NBP API is public
        requestDefaults: {
            baseURL: 'https://api.nbp.pl/api',
            headers: { Accept: 'application/json' },
        },
        properties: [/* defined below */],
    };
}
```

**IMPORTANT:** Do NOT include a `credentials` property at all. NBP is a public API.

**Resource selector** with 2 options:
- `Exchange Rate` (value: `exchangeRate`)
- `Gold Price` (value: `goldPrice`)

**Exchange Rate operations** (6 operations, all displayOptions show resource=exchangeRate):

1. **Get Current Rate** (value: `getCurrentRate`)
   - Parameters: `table` (options: A/B/C, default A), `currencyCode` (string, required, default 'EUR', placeholder 'EUR')
   - Routing on currencyCode: `url: '=/exchangerates/rates/{{$parameter["table"]}}/{{$value}}/'`, method: GET

2. **Get Rate by Date** (value: `getRateByDate`)
   - Parameters: `table`, `currencyCode`, `date` (string, required, placeholder 'YYYY-MM-DD', description includes "Format: YYYY-MM-DD")
   - Routing on date: `url: '=/exchangerates/rates/{{$parameter["table"]}}/{{$parameter["currencyCode"]}}/{{$value}}/'`, method: GET

3. **Get Rate Date Range** (value: `getRateDateRange`)
   - Parameters: `table`, `currencyCode`, `startDate` (string, required, placeholder 'YYYY-MM-DD'), `endDate` (string, required, placeholder 'YYYY-MM-DD', description: "Maximum 93-day range")
   - Routing on endDate: `url: '=/exchangerates/rates/{{$parameter["table"]}}/{{$parameter["currencyCode"]}}/{{$parameter["startDate"]}}/{{$value}}/'`, method: GET

4. **Get Last N Rates** (value: `getLastNRates`)
   - Parameters: `table`, `currencyCode`, `count` (number, required, default 10, description: "Number of last published rates to retrieve")
   - Routing on count: `url: '=/exchangerates/rates/{{$parameter["table"]}}/{{$parameter["currencyCode"]}}/last/{{$value}}/'`, method: GET

5. **Get Current Table** (value: `getCurrentTable`)
   - Parameters: `table`
   - Routing on table: `url: '=/exchangerates/tables/{{$value}}/'`, method: GET

6. **Get Table by Date** (value: `getTableByDate`)
   - Parameters: `table`, `date` (string, required, placeholder 'YYYY-MM-DD')
   - Routing on date: `url: '=/exchangerates/tables/{{$parameter["table"]}}/{{$value}}/'`, method: GET

**Gold Price operations** (4 operations, all displayOptions show resource=goldPrice):

1. **Get Current Price** (value: `getCurrentPrice`)
   - No additional parameters
   - Routing on resource or operation: `url: '/cenyzlota/'`, method: GET
   - Use a hidden/fixed property or put routing on the operation option itself

2. **Get Price by Date** (value: `getPriceByDate`)
   - Parameters: `goldDate` (string, required, placeholder 'YYYY-MM-DD')
   - Routing on goldDate: `url: '=/cenyzlota/{{$value}}/'`, method: GET

3. **Get Price Date Range** (value: `getPriceDateRange`)
   - Parameters: `goldStartDate` (string, required, placeholder 'YYYY-MM-DD'), `goldEndDate` (string, required, placeholder 'YYYY-MM-DD', description: "Maximum 93-day range")
   - Routing on goldEndDate: `url: '=/cenyzlota/{{$parameter["goldStartDate"]}}/{{$value}}/'`, method: GET

4. **Get Last N Prices** (value: `getLastNPrices`)
   - Parameters: `goldCount` (number, required, default 10)
   - Routing on goldCount: `url: '=/cenyzlota/last/{{$value}}/'`, method: GET

**Parameter naming strategy:** Use prefixed names for Gold Price params (`goldDate`, `goldStartDate`, `goldEndDate`, `goldCount`) to avoid name collisions with Exchange Rate params. Each parameter must have `displayOptions: { show: { resource: ['goldPrice' or 'exchangeRate'], operation: ['specificOp'] } }`.

**Table parameter:** Shared across all Exchange Rate operations. Define once with `displayOptions: { show: { resource: ['exchangeRate'] } }`. Put routing on the last unique parameter of each operation (the one that completes the URL).

**For Get Current Price (no params):** Create a routing property on the operation option itself using `routing` on the operation options array entry, OR add a hidden dummy property. Best approach: put `routing: { request: { method: 'GET', url: '/cenyzlota/' } }` directly on the `getCurrentPrice` option in the operation options array (n8n supports this).

**Each operation option** must have `name`, `value`, `description`, and `action` fields.

**Date fields use `type: 'string'`** (not `dateTime`) because NBP API expects YYYY-MM-DD format and dateTime may send full ISO timestamps. Use `placeholder: 'YYYY-MM-DD'` on all date fields.
