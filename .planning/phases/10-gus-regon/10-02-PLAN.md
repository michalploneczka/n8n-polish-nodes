**execute() method** -- iterate items, switch on resource/operation:

For `company` resource:
- **searchByNip**: Get `nip` param, build URL from credentials environment, build envelope with `daneSzukajPodmiotyEnvelope(url, { nip })`, call `gusRegonApiRequest.call(this, envelope)`, parse with `parseSearchResponse(response)`, push each result as `{ json: result as IDataObject }`
- **searchByRegon**: Same but with `regon` param and `{ regon }` in envelope
- **searchByKrs**: Same but with `krs` param and `{ krs }` in envelope
- **getFullData**:
  1. Get `regonForReport` and `includePkd` params
  2. Build URL from credentials environment
  3. First, search by REGON to determine entity type: `daneSzukajPodmiotyEnvelope(url, { regon: regonForReport })`, call, parse with `parseSearchResponse`
  4. Get `Typ` field from search result (P = legal entity, F = natural person)
  5. Select main report type: `Typ === 'P'` -> `REPORT_TYPES.legalEntity`, `Typ === 'F'` -> `REPORT_TYPES.naturalPerson`, else fallback to `REPORT_TYPES.legalEntity`
  6. Call `danePobierzPelnyRaportEnvelope(url, regonForReport, mainReportType)`, call `gusRegonApiRequest`, parse with `parseReportResponse`
  7. If `includePkd` is true, determine PKD report type: `Typ === 'P'` -> `REPORT_TYPES.legalEntityPkd`, `Typ === 'F'` -> `REPORT_TYPES.naturalPersonPkd`
  8. Call PKD report, parse with `parseSearchResponse` (PKD returns array of activity records)
  9. Merge: `{ ...mainReport, pkdCodes: pkdResults }` as the final JSON output
  10. Push result as `{ json: mergedResult as IDataObject }`

**Error handling:** Wrap the per-item block in try/catch. If `this.continueOnFail()` returns true, push `{ json: { error: (error as Error).message } }` and continue. Otherwise re-throw.

**Helper for URL resolution:** Inside execute(), before the loop, get credentials once and resolve URL:
```typescript
const credentials = await this.getCredentials('gusRegonApi');
const environment = credentials.environment as string;
const url = environment === 'production' ? URLS.production : URLS.test;
```

Note: `gusRegonApiRequest` also gets credentials internally for login. The URL is needed in execute() to build the operation-specific SOAP envelopes with `<wsa:To>`. This is intentional -- envelopes need the URL for the `wsa:To` header, while GenericFunctions uses it for the HTTP request URL and login.

**Return:** `return [returnData];`
