**Test helper:**
```typescript
import { createMockExecuteFunctions } from '@n8n-polish-nodes/test-utils';
import { Ceneo } from '../nodes/Ceneo/Ceneo.node';

const TEST_API_KEY = 'test-ceneo-api-key';
const TEST_TOKEN = 'test-bearer-token-123';

function createCeneoMock(params: Record<string, unknown>) {
  const mock = createMockExecuteFunctions(params);
  mock.getCredentials = jest.fn().mockResolvedValue({
    apiKey: TEST_API_KEY,
  });
  mock.helpers.httpRequest = jest.fn();
  return mock;
}
```

**Important:** Since GenericFunctions uses module-level `cachedToken`, import and call `resetTokenCache` in `beforeEach` to ensure clean state:
```typescript
import { resetTokenCache } from '../nodes/Ceneo/GenericFunctions';

  resetTokenCache();
});
```

**Mock httpRequest strategy:** Each v3 test needs TWO httpRequest calls -- first for GetToken (returns TEST_TOKEN string), second for the actual endpoint. Each v2 test needs ONE httpRequest call (v2 uses apiKey directly, no token).

**Test cases (8 total):**

1. **"should have correct node metadata"** -- verify `node.description.displayName === 'Ceneo'`, `node.description.name === 'ceneo'`, resource options length === 3, credentials[0].name === 'ceneoApi'

2. **"should get token from AuthorizationService"** -- mock httpRequest to return TEST_TOKEN on first call, then mock product data on second call. Call execute with `resource: 'product', operation: 'getTopCategoryProducts', categoryName: 'Laptopy', top: 10`. Verify first httpRequest call used URL containing `AuthorizationService.svc/GetToken` and header `Authorization: \`Basic ${TEST_API_KEY}\``

3. **"should get top category products (v3)"** -- mock httpRequest: first call returns TEST_TOKEN, second returns `[{ productId: 1, name: 'Laptop X', price: 2999.99 }]`. Call with `resource: 'product', operation: 'getTopCategoryProducts', categoryName: 'Laptopy', top: 10`. Verify second httpRequest URL contains `/api/v3/GetTopCategoryProducts`, qs contains `{ categoryName: 'Laptopy', top: 10 }`, and header contains `Bearer ${TEST_TOKEN}`. Verify returnData has 1 item with productId: 1.

4. **"should get all offers (v2)"** -- mock httpRequest to return `{ offers: [{ offerId: 100, price: 49.99 }] }`. Call with `resource: 'product', operation: 'getAllOffers', shopProductIds: '123,456'`. Verify httpRequest URL contains `/api/v2/function/webapi_data_critical.shop_getProductOffersBy_IDs/Call`, qs contains `apiKey: TEST_API_KEY`, `resultFormatter: 'json'`, `shop_product_ids: '123,456'`, method is POST. Verify returnData has response data.

5. **"should get top 10 cheapest offers (v2)"** -- mock httpRequest to return `{ offers: [{ offerId: 200, price: 29.99 }] }`. Call with `resource: 'product', operation: 'getTop10CheapestOffers', shopProductIds: '789'`. Verify URL contains `shop_getProductTop10OffersByIDs`.

6. **"should list categories (v3)"** -- mock httpRequest: first returns TEST_TOKEN, second returns `[{ categoryId: 1, name: 'Elektronika' }]`. Call with `resource: 'category', operation: 'list'`. Verify URL contains `/api/v3/GetCategories`.

7. **"should get execution limits (v2)"** -- mock httpRequest to return `{ dailyLimit: 1000, remaining: 950 }`. Call with `resource: 'account', operation: 'getLimits'`. Verify URL contains `GetExecutionLimits`.

8. **"should handle API errors as NodeApiError"** -- mock httpRequest to throw `new Error('Unauthorized')`. Call with any operation. Verify the execute throws and error message contains 'Ceneo API'. Use `expect(node.execute.call(mock)).rejects.toThrow()`.
